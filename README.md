## Projects

**[hackerrank-orchestrate-june26](https://github.com/simrun09/hackerrank-orchestrate-june26)** — Python, Mistral Pixtral, multimodal JSON mode

A submission for the June 2026 HackerRank Orchestrate hackathon. The task
was to build a system that reviews damage claims (for cars, laptops, or
packages) by looking at submitted images alongside the customer's chat
transcript, their claim history, and the object's evidence rules, and
deciding whether the claim is supported, contradicted, or unverifiable,
plus nine supporting fields. My solution uses one multimodal call per row
(44 rows total) with a validation step at the end that keeps every field
inside its allowed vocabulary, and treats every word in the transcript and
any text visible in the images as data rather than instructions. Built solo
in about 20 hours and documented across seven phase-commits. Full notes on
the design decisions, tradeoffs, and honest weaknesses are in
[`MY_BUILD.md`](https://github.com/simrun09/hackerrank-orchestrate-june26/blob/main/MY_BUILD.md).

**[project-inception](https://github.com/simrun09/project-inception)** — Claude skill

A Claude skill that helps set up a project before any code is written. You
give it a rough idea, answer a few questions, and it walks through problem
definition, brief research, a feasibility check with a small test, and a
technical spec — then generates the working-memory files a coding agent
uses to build from. I built it because I found myself jumping into code
before the problem was properly defined, and wanted a more structured
starting point for future projects.
